# Тест-кейсы для микросервиса объявлений

**Host:** https://qa-internship.avito.com

| ID | Название проверки | Описание и Шаги | Ожидаемый результат | Приоритет |
|:---|:---|:---|:---|:---|
| **TC-01** | **Создание объявления (Успешный сценарий)** | 1. Отправить `POST /api/1/item` с валидным телом запроса (sellerID, name, price, statistics). | Статус **200 OK**. В ответе приходит подтверждение сохранения ("Сохранили объявление") и ID объявления. | **High** |
| **TC-02** | **Создание объявления без обязательного поля** | 1. Подготовить тело запроса без поля `price`.<br>2. Отправить `POST /api/1/item`. | Статус **400 Bad Request**. Объявление не создается. | **Medium** |
| **TC-03** | **Получение объявления по ID** | 1. Создать объявление и получить его ID.<br>2. Отправить `GET /api/1/item/{id}`. | Статус **200 OK**. Данные в ответе (name, price, sellerId) совпадают с теми, что были отправлены при создании. | **High** |
| **TC-04** | **Получение объявления по несуществующему ID** | 1. Сгенерировать случайный валидный UUID, которого нет в базе.<br>2. Отправить `GET /api/1/item/{uuid}`. | Статус **404 Not Found**. | **Medium** |
| **TC-05** | **Получение всех объявлений продавца** | 1. Создать объявление для конкретного `sellerID`.<br>2. Отправить `GET /api/1/{sellerID}/item`. | Статус **200 OK**. В ответе приходит список, содержащий созданное объявление. | **High** |

| **TC-06** | **Получение статистики объявления** | 1. Создать объявление с заполненной статистикой.<br>2. Отправить `GET /api/1/statistic/{id}`. | Статус **200 OK**. Поля статистики (likes, viewCount, contacts) совпадают с отправленными. | **Medium** |
